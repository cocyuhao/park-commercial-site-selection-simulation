# Codex 主线防偏航规则

> 当前口径：这是插在仿真主线里的保护层，不是离开主线去做另一件事。

## 为什么要有这一层

本项目已经进入长上下文、多 agent、多轮同步状态。风险不是“少写一个总结”，而是新对话或新 agent 容易把旧完成度、旧分数、旧 DeepSeek 草稿、同事辅助代码和老板方法资料混成一团，导致主线重新跑歪。

因此 Codex 自身强化只允许做能直接保护主线的事情：

- 把当前最高优先级写清楚。
- 把旧口径和新口径的冲突写清楚。
- 把下一步可执行动作写清楚。
- 把启动检查、门禁和证据路径做成可运行入口。
- 完成后立即回到当前主线：全局 AI 仿真决策系统的对象链路、AI 工作台、资料/方法对象池、仿真任务和旧产物信任分级。

## 当前主线

当前主线不是地图，也不是泛泛 UI，也不是继续写学习总结。

当前主线是：

1. 总定位是“AI 驱动仿真决策系统”；公园商业选址只是当前业务场景，不得把系统写死为单一公园商业工作台。
2. 继续尊重老板六份方法资料触发的系统级重基线。
3. 保持 DeepSeek 低成本语义工人定位。
4. 保持 `choice_probability` 和 `simulation_validation_target` 均为 `needs_review`。
5. 方法、工具、插件、论文和同事成果必须进入审计清单，不能再用一句“已学习/已参考/已使用”带过。
6. 已落地的对象池只是第一处能力验证；后续要继续重构 AI 工作台、资料池/方法对象池、仿真任务入口和报告链路。
7. 不允许旧 dry-run、旧分数、旧报告或旧 UI 文案被误写成完整仿真。
8. 对历史长文件夹建立信任分级：仍可信、需降级、仅历史参考、应废弃，不能默认沿用旧结论。

## 新对话启动顺序

新对话或新 agent 接入时，优先运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\00_control\start_codex_mainline.ps1
```

如果要做完整门禁：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\00_control\start_codex_mainline.ps1 -FullGate
```

脚本会生成或刷新：

- `40_quality_evidence/codex_mainline_context_20260604.json`
- `40_quality_evidence/codex_mainline_context_20260604.md`

## 必须避免的跑偏

- 不要把 Codex 自身强化当作新主线。
- 不要再写“参考过论文”式空总结。
- 不要再用“调用过插件/装过工具”替代落点、风险和决策审计。
- 不要把老板资料当补缺口参考；它是方法重基线。
- 不要让 DeepSeek 决定最终仿真、最终排名、ROI 或 checked 证据。
- 不要把旧 `P4 完整仿真`、旧裸分数、旧 dry-run 当完成。
- 不要完全同步同事远端覆盖本地成果；只读比较、选择吸收。
- 不要因为对象池已能增删改查就宣称全局系统完成；它只是新对象链路的第一块。
- 不要把“门禁”当用户界面文案；门禁是内部质量机制，用户界面要说“复核、缺口、依据、推进动作”。

## 当前立即下一步

完成防偏航层验证后，继续实现：

- 全局推进台继续从旧项目总览过渡到对象链路状态。
- 专家 AI 工作台按项目综合、节点会话、历史、新对话、生成报告重构。
- 资料池和方法对象池统一采用、放弃、删除、锁定、解析说明和关联对象。
- 仿真任务入口显示输入对象、校准目标、运行状态和待复核结果。
- 建立历史产物信任地图，防止旧文件和新方向互相误判。
- API、Selenium/Browser、人类视觉验证和落实性门禁同步更新。
