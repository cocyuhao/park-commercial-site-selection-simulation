# 方法论

## 总体原则

先证据，后模型；先底稿，后汇报；先可解释，后复杂化。

## 1. 数据证据流

所有关键数字进入 `40_quality_evidence/evidence_ledger.csv`。

字段包括：

- 指标 ID
- 指标名称
- 数值
- 单位
- 来源文件
- 页码或幻灯片号
- 原文片段
- 抽取方式
- 置信度
- 校验状态

证据强度默认分级：

| 证据类型 | 默认强度 | 说明 |
|---|---|---|
| 用户提供的原始经营数据 | 高 | 票务、收入、租金、分成等一手数据 |
| PDF 报告正文/表格 | 中高 | 需要记录页码和原文 |
| 高德 POI/路径数据 | 中高 | 需要记录抓取时间、API、参数 |
| 官方公开资料 | 中 | 需要记录网址和访问日期 |
| PPT 方案页 | 中低 | 默认视为假设或表达材料 |
| 模型估算 | 估算 | 必须标注参数和假设 |

## 2. 需求建模

游客需求由四类信号组成：

```text
需求强度 = 客流规模 × 客群匹配 × 消费偏好 × 场景触发
```

关键输入：

- 客流规模：日均、小时、节假日、季节。
- 客群结构：亲子、运动、白领、游客、老人。
- 消费偏好：TGI、品牌到访、热门 POI、客单价。
- 场景需求：咖啡、亲子餐、夜间餐酒、文创、运动补给、便利零售。

后续模型必须显式考虑“人”的概率差异，而不是只按总客流做静态计算：

- 游客分群：亲子、运动、白领、老人、外地游客、夜间消费人群等。
- 个体属性：同行人数、预算、时间压力、耐心、体力、儿童需求、对品牌/价格敏感度。
- 场景触发：天气、节假日、活动、时间段、排队、封路、临时摊位、突发购买念头。
- 概率输出：业态选择概率、店铺选择概率、转化率、放弃率、外溢率和停留时间分布。
- 用户临时提出的个性化需求先进入“假设池”，只作为待验证场景，不直接变成事实参数。

## 3. 供给建模

供给不是简单 POI 数量，而是“可达的有效供给”。

```text
有效供给 = 同业态 POI × 距离衰减 × 质量权重 × 营业时段匹配 × 容量估计
```

关键规则：

- 园内和园外供给分开计算。
- 步行 3/5/10 分钟圈层优先于直线距离。
- 高德 typecode 要映射为项目自定义业态。
- 评分、人均、品牌、营业状态能获取时应进入质量权重。

## 4. 供需缺口识别

基础公式：

```text
缺口分数 =
需求强度 × 客流热度 × 场景匹配度 × (1 - 供给覆盖率) × 外溢风险 × 落地可行性
```

输出等级：

| 等级 | 含义 | 行动 |
|---|---|---|
| P0 | 高需求、低供给、高外溢、可落地 | 优先改造或招商 |
| P1 | 有明显需求但依赖更多数据 | 作为第二批方案 |
| P2 | 需求存在但收益或落地不确定 | 保留观察或小规模试点 |

## 5. 仿真和优化

按复杂度递进：

1. Level 1：供需缺口计算器。
2. Level 2：Huff/重力模型 + 人群概率原型，估算不同游客分群选择某业态/某店的概率。
3. Level 3：最大覆盖选址模型，选择有限点位覆盖最多需求。
4. Level 4：游客 Agent 仿真，模拟路径、消费、排队、放弃、外溢和个性化突发需求。
5. Level 5：财务场景和压力测试。

仿真计算由本地 Python 执行；DeepSeek 只辅助生成场景解释、需求假设草稿和报告初稿。任何概率、收益、排序和最终推荐都必须能追溯到参数表、证据 ID 或明确人工假设。

所有概率仿真必须记录：

- 输入参数版本。
- 随机种子。
- 仿真样本数。
- 保守/基准/乐观场景。
- 主要输出的均值、分位数或置信区间。

## 6. 财务测算

公园端收益口径：

```text
公园端净收益 =
新增捕获客流 × 转化率 × 客单价 × 公园分成比例
+ 固定租金
- 改造投入折算
- 运营和管理成本
- 季节/天气/审批风险折减
```

必须输出保守、基准、乐观三档。

## 7. 质量门禁

进入下一阶段前必须满足：

- 数据来源记录完整。
- 关键指标单位明确。
- 异常值和空值有处理策略。
- 模型参数有来源或明确标注为假设。
- 输出结论可被证据表追溯。

## 8. 专家网站交付方法

最终网站不是宣传页，而是行业专家使用的决策界面。网站必须服务于“复核输入、调整场景、理解模型、追溯证据、导出结论”五件事。

核心原则：

- 证据优先：每个推荐、风险、收益和排序都要能回到证据 ID、POI 核验记录、参数表或明确假设。
- 模型透明：展示关键输入、权重、概率、随机种子、样本数、场景名称和不确定性区间。
- 专家可调：允许专家调整人群比例、转化率、租金、分成、天气/节假日/活动等场景参数，并看到结果变化。
- 视觉有用：地图、流线、热力、图表和微动效只用于解释空间、时间、概率和风险，不做无关装饰。
- AI 有边界：DeepSeek 可辅助生成解释、提示风险和整理个性化需求草稿，但网站上的最终结论必须来自 Python 计算、证据链和人工门禁。

网站验收时至少检查：

- 推荐点是否能追溯到证据和参数。
- 场景切换是否保留原始参数和输出摘要。
- 地图、图表、表格是否能在桌面和移动端清晰阅读。
- 专家批注和人工调整是否记录理由。
- 导出的报告、表格和 API 验收附件是否与当前模型版本一致。
