## 1. 当前真正主线

本项目不是“修一个旧 P6 网页”，而是建设 **AI 驱动仿真决策系统**。当前业务场景是公园商业选址，但系统主线是全局的：

资料证据 -> 人群潜在状态 -> 行为程序 -> 选择概率 -> 空间语境 -> 验证目标 -> 仿真预检 -> 节点推进 -> AI 共识 -> 报告工作稿

这条链来自：

- 老板六份方法资料：要求 `state -> behavior -> choice -> validation`，旧 POI/TGI 裸分数不能当最终决策。
- 2026 仿真/agentic UI 学习：要求 human oversight、检查点、对象可采用/放弃/锁定、不要让 LLM 直接编最终结论。
- 当前工程底座：FastAPI dashboard/object-chain、资料池状态、仿真对象池、DeepSeek-only 生产边界、预检门禁和 QA 证据链。

## 2. 当前这段工作在主线里的位置

本轮修“工作流导航、节点详情重复新增、静态版本和证据脱敏”不算主线产物本身，只算 **迁移护栏**。

它有价值的前提是：

- 防止旧 P6 壳把新系统拖回并列模块。
- 防止旧交互残留污染新工作流。
- 防止浏览器旧缓存和证据泄露让验证失真。
- 把人类路径问题写入门禁，避免只靠静态检查。

它不能继续无限扩展。旧问题只有在会污染新主线时才处理；纯旧页面美化、旧文案边角、旧结构微调一律进入后续遗留清单，不占主线。

## 4. 防循环停止规则

以下任一情况出现，必须停止修旧壳，回到主线：

1. 连续两个改动只是在修旧页面边角，而没有推进对象链、资料/空间底座、仿真对象或预检。
2. 某个 bug 不能映射到 `保留 / 重构 / 隐藏 / 废弃`。
3. 改动只让旧页面“看起来更顺”，但没有改善用户完成主线任务的能力。
4. 验证只证明按钮存在、文件存在或截图存在，没有覆盖人类路径。
5. 新增文档只在解释“为什么”，没有落到代码、门禁或下一步实现。
