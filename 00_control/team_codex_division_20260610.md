# 双 Codex 现代化分工（2026-06-10）

## 分工原则

本项目当前以本地这份工作区为主线。另一位同事也有 Codex，但不再走传统“你写前端我写后端”的粗分工，而是按风险和上下文深度分：

- 本机主 Codex：负责模型、知识库、DeepSeek/国产闭源模型约束、仿真架构、报告口径、证据链、门禁和最终合并。
- 同事 Codex：负责低风险、可视觉验证、可独立回滚的前端细节和客服端易用性，不碰核心模型和结论链路。

## 同事可接任务

1. 前端细节修复
   - 检查按钮文字是否溢出。
   - 检查输入框、输出框、资料池抽屉、折叠区在 1366、1440、1920、移动宽度下是否舒适。
   - 把明显过密的说明文案改短，但不得改业务结论。

2. 视觉 QA
   - 用 Browser/Chrome/Selenium 跑 AI 工作台、项目总览、资料池、报告页。
   - 每个问题给截图、URL、操作路径和可复现步骤。
   - 不直接改模型、报告生成提示词、知识库脚本。

3. 资料池客服化
   - 空态、加载态、错误态、删除确认、采用/暂不采用按钮文案。
   - 禁止出现 raw、payload、debug、traceback、API response 等内部词。

4. 轻量无障碍与交互
   - 给关键按钮补稳定 `data-testid` 或明确文本。
   - 检查键盘 Tab 顺序和 focus 样式。
   - 不重写页面架构。

如果必须碰上述区域，先提交独立分支和说明，不直接覆盖主线。

## 同事交付格式

同事每次只交一个小 PR 或小 commit，包含：

- 修改点列表
- 浏览器截图路径
- Selenium/Playwright 验证 JSON
- 仍未解决的问题
- 是否碰到核心文件
