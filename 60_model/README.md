# 模型目录

本目录保存需求模型、供给模型、缺口计算、仿真和优化。

子目录：

- `configs/`：业态映射、参数模板、场景配置。
- `notebooks/`：探索分析笔记本。
- `src/`：可复用 Python 模块。
- `scenarios/`：不同项目或压力测试场景。
- `validation/`：模型校验结果。

建模顺序：

1. Level 1：供需缺口计算器。
2. Level 2：Huff/重力模型 + 人群概率原型。
3. Level 3：最大覆盖选址优化。
4. Level 4：游客 Agent 仿真。
5. Level 5：财务压力测试。

不要在没有数据校准时直接跳到复杂仿真。

计划边界：

- P2 先做本地 Python 可复跑概率原型，显式记录游客分群、需求触发、选择概率、随机种子和场景参数。
- DeepSeek 只辅助整理个性化需求、场景草稿和解释文本，不直接决定概率、收益或候选点排序。
- P4 再做完整游客 Agent 仿真，模拟路径、停留、消费、排队、放弃和外溢。
- Postman 从 P2 开始只作为 API 契约和 smoke/regression 测试规划，P4 再扩展到仿真 API 回归集合。
