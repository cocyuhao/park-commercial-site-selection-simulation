## 4. 老板模型必须这样落地

来源：

- `10_research/boss_method_materials_20260604/boss_model_inventory_20260604.md`
- `10_research/boss_method_materials_20260604/unified_simulation_method_matrix_20260604.md`
- `10_research/boss_method_materials_20260604/method_absorption_landing_register_20260604.md`

落地规则：

- HumanLM：报告要先写人群潜在状态，不只写“亲子/白领/游客”标签。
- ROTE：把行为写成可编辑程序：触发、路径、停留、消费、放弃、外溢。
- RL + LLM 双重奖励：当前不训练 PPO，但报告要有“微观行为合理性 + 宏观数据一致性”的判断结构。
- SSR / FLR：不直接打分；先写自然语言理由，再映射为推进优先级。
- Agent Bank：人物画像不能凭空捏，要由 PDF/TGI/POI/策划材料形成弱记忆。
- Huff / Logit / Gravity：用于解释吸引力、距离、供给替代和选择概率，但不能伪装成已校准最终概率。

## 5. 下一版报告必须改变的结构

旧结构问题：边界和补证太重。

新结构建议：

1. 结论摘要：基于已给材料，现在最可能成立的 3-5 个经营判断。
2. 数据预测底座：客流峰值、TGI、热门 POI、客单线索、策划面积和节点功能。
3. 人群与行为预测：按 HumanLM + ROTE 写出人群状态、行为程序和消费触发。
4. 六节点调整方案：每个节点给出原计划判断、保留项、降级项、替换项、分期动作、预测客群、价格带、运营时段、风险控制。
5. 组合策略：南门簇、北园节点、康养节点、重资产节点如何分组推进。
6. 试运营设计：用已有数据先跑哪些小实验，如何看结果。
7. 收益与价格带：只做区间和情景，不做伪精确 ROI。
8. 仍需复核：放最后，作为风险附录，不再喧宾夺主。
