## 0. 先修正判断

这六份资料与最终目标强相关。它们不只是“可以参考的论文”，而是在回答这个系统最核心的几个问题：

- 人的行为能不能被模拟？
- LLM 应该放在仿真链路的哪一层？
- 大规模仿真怎样避免变成批量编故事？
- 分数、优先级、购买意向、节点建议应该怎样从“人的表达”转成“决策指标”？
- 什么时候可以说“可用”，什么时候只能说“早期信号 / 待复核”？

我刚才提前落了若干草稿文件，这是节奏过快。后续实现必须以本文和后续补充论文矩阵为准，已经新增的草稿只能作为待审输入，不代表已完成设计。

## 2. ROTE：行为不是散文，而是程序

资料：`2510.01272v1.pdf`

核心方法：

- 论文把行为理解为“behavioral programs”，而不是单次语言回答。
- LLM 的作用是生成候选行为程序，不是直接决定最终行为。
- 行为程序可以是有限状态机式结构：状态、观察、动作、转移。
- 观察到新的行为轨迹后，用 Bayesian inference / Sequential Monte Carlo 更新候选程序权重。
- 它的价值在于可解释、可复用、可组合，比每次重新让 LLM 猜测更稳。

迁移到公园商业选址：

- “亲子家庭会买轻餐”不够，必须拆成程序：
  - 进入公园
  - 儿童疲劳 / 排队 / 天气热
  - 搜索最近遮阴和饮品点
  - 判断距离、排队、价格、座位
  - 消费、放弃或外溢到园外
- 节点不应该只有分数，应展示“行为程序为什么会把人带到这里 / 为什么会放弃”。
- 后续如果有真实路径、排队、停留时长，就能反过来更新行为程序权重。

不能照搬：

- ROTE 示例多是高层动作预测，不是完整商业收益模型。
- 它需要观察轨迹，否则程序权重只是候选假设。
- 不能把 DeepSeek 生成的程序直接当真实人群行为。

## 3. HumanLM：不要模仿表面回答，要对齐潜在状态

资料：`2603.03303.pdf`

核心方法：

- 论文批评了 response imitation：只模仿用户话术、表情、长短，很容易错过真正立场。
- 它把用户反应拆成多个 latent state dimensions：
  - belief
  - goal
  - value
  - stance
  - emotion
  - communication
- 模型先生成潜在状态，再合成回答。
- 用 LLM judge 比较一批 latent states 的对齐程度，再把分数作为强化学习奖励。
- 实验说明，同一个人也不会两次说出完全一样的话，所以“字面相似”不是好目标。

迁移到公园商业选址：

- 我们的人群画像不能只写“白领、老人、亲子、游客”，要有潜在状态：
  - 目的：打卡 / 休息 / 健身 / 社交 / 陪伴孩子
  - 压力：时间压力、预算压力、返程压力
  - 情绪/身体状态：疲劳、饥饿、炎热、焦虑、兴奋
  - 价值/偏好：安全、价格、品牌信任、便利、体验感
  - 沟通/决策方式：愿意探索、保守、跟随同伴、看评价
- DeepSeek 可用于生成“潜在状态解释”和“人类语言反馈”，但必须再进入本地规则/验证层。

不能照搬：

- HumanLM 的目标是模拟文本回应，不是直接模拟空间路径和消费。
- 论文依赖训练/对齐流程；我们现在没有真实问答或访谈数据，不能声称已达到它的效果。
- 对我们来说，它更像“状态建模原则”，不是可直接搬来的模型。

## 4. 社区高精度仿真：LLM 不是主引擎，而是奖励/合理性层

资料：`人工智能模拟实验论文.docx`、`已转换 - main-1.docx`

核心方法：

- 论文构建真实社区的高保真环境，使用真实 GIS/BIM/人口数据。
- 生成 3000 个 AI agents。
- 使用 RL + LLM 混合框架：
  - RL 保证行动策略可训练、可复跑。
  - LLM 提供微观行为合理性评价。
  - 宏观层面用真实手机使用数据对齐。
- 双重奖励：
  - 微观：LLM 判断个体行为是否符合社会常识。
  - 宏观：模拟手机使用曲线是否接近真实手机数据。
- 验证不是只看“像不像”，而是用了多层指标：
  - 一周曲线相关性超过 90%。
  - SARIMA 检查真实/模拟时间序列内在一致性。
  - SSIM = 0.79。
  - Phase Shift = -1.48h / -0.06h。
  - KL-Divergence = 0.071。
  - DTW-R2 = 0.99。
  - 睡眠模式、日常活动分布是否符合社会常识。

迁移到公园商业选址：

- 我们必须建立宏观校准目标，而不只是跑一批 agent：
  - 分入口 / 分时段客流曲线
  - 节点停留时长
  - 路径通过量
  - 业态转化率
  - POI 供给密度
  - 节假日、天气、活动场景的峰谷变化
- DeepSeek 可以做：
  - 微观行为合理性评价
  - 行为程序草稿
  - 缺口解释
  - 报告语言
- 本地 Python 必须做：
  - 抽样
  - 路径/距离/排队/容量
  - 敏感性分析
  - 复跑
  - 指标计算

不能照搬：

- 我们现在没有 Unreal Engine 高保真环境，也没有完整 GIS/BIM。
- 现有项目 DWG 仍未完全转换，不能做精确空间仿真结论。
- 没有真实入口客流、转化率、收益成本、授权数据前，不能宣称 P4 完整仿真完成。

## 5. GameLook / SSR / Agent Bank：不要让模型直接打分

资料：长图与 `591775 (已转换).pdf`

核心方法：

- 文章引用的“1000 人生成式智能体模拟”强调：
  - 不能只用人口统计信息或浅层 persona。
  - 用长访谈记录作为 agent memory，效果显著更好。
  - 专家反思可作为中间记忆。
  - 问卷准确率 85%，人格相关性 80%，经济博弈相关性 66%。
  - 人类两周后重做同一问卷准确率约 81%，说明“人类反馈本身也有波动”。
- SSR 方法强调：
  - 不要让 LLM 直接输出 1-5 分，容易集中在 3 分。
  - 先让合成消费者自然语言表达购买意向。
  - 再把自然语言用语义相似度映射到李克特量表参考句。
  - 可保留理由说明，避免只剩数字。
- 产品验证强调：
  - 和 Sensor Tower、真实 A/B 测试、人类反馈对比。
  - 合成用户适合早期方向预警，不替代最终验证。

迁移到公园商业选址：

- 节点页不应裸露“分数”，更应该展示：
  - 这个节点吸引哪类人
  - 人为什么会来
  - 什么情况下会消费
  - 什么情况下会放弃
  - 需要补什么证据
  - 映射出来的优先级是什么
- “分数”若保留，应变成次级信息，且要写清：
  - 它来自哪些语言判断
  - 映射规则是什么
  - 缺什么数据会改变它
- 更适合的表达可能是：
  - 优先推进
  - 待补资料后再判断
  - 只做空间观察
  - 暂缓

不能照搬：

- 游戏素材、应用图标、购买意向与公园选址不同。
- 早期合成用户信号不能替代真实客流、场地、授权和运营财务。
- 如果没有访谈/行为记忆，只用“亲子家庭”“白领”这类标签会很薄。

## 6. 对 DeepSeek 接入的修正判断

DeepSeek 便宜，应该接，但位置必须谨慎。

可行位置：

- 批量生成候选行为程序。
- 把资料拆成潜在状态。
- 用自然语言表达“消费/放弃/外溢理由”。
- 把本地仿真结果写成业务报告。
- 作为 LLM judge 对行为合理性做草稿评分。

不该做：

- 不应逐游客实时调用 DeepSeek。
- 不应由 DeepSeek 直接给最终节点排名。
- 不应由 DeepSeek 生成收益预测或 ROI。
- 不应把 DeepSeek 输出写入 checked 证据。
- 不应让 DeepSeek 在缺真实校准数据时声称仿真完成。

推荐架构：

1. DeepSeek 生成“行为程序候选”和“潜在状态候选”。
2. 本地规则把候选转成可执行参数。
3. Python 本地跑大规模抽样 / ABM / Monte Carlo。
4. 用真实数据校准宏观指标。
5. DeepSeek 只负责解释结果、指出缺口、生成报告草稿。
6. Codex / 人工复核决定是否进入 checked 或正式报告。

## 7. 对产品界面的直接影响

这些材料会改变页面，而不只是算法：

- “分数”要降级，优先级和建议要升级。
- 每个节点要有“人为什么来 / 为什么买 / 为什么不买 / 缺什么证据”的解析。
- 资料池要能显示哪些资料被用于人群、行为、空间、供给、校准。
- 用户应能看见并修改采用的人群与假设，否则仿真是黑箱。
- AI 工作台不能只聊天，要能沉淀：
  - 人群状态
  - 行为程序
  - 数据缺口
  - 报告草稿
  - 待复核判断
- 报告要把“早期信号”和“可确认事实”分开。
